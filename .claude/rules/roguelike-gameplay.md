# Roguelike Survival Gameplay Rules

## 核心循环

玩家移动 -> 自动攻击 -> 击杀怪物 -> 掉落 XP -> 升级三选一 -> 构筑增强 -> 更高压力波次。

## 数据驱动

- 武器、被动、升级、怪物、波次都必须配置化。
- 不要把数值硬编码在 System 内部。
- System 读取配置并执行逻辑。
- 新增配置必须包含默认值和调试说明。

## 武器结构

每个武器至少考虑：

- id、名称、稀有度、标签。
- 冷却、伤害、范围、持续时间、投射物数量。
- 目标选择方式。
- 命中规则。
- 升级成长。
- PixiJS 程序化 VFX。

## 怪物结构

每个怪物至少考虑：

- id、类型、血量、速度、碰撞半径、接触伤害。
- XP 掉落、权重、生成时间窗。
- PixiJS 程序化外观。
- 是否精英、是否 Boss、是否特殊行为。

## 升级

- 升级选项由权重池生成。
- 升级效果必须写成可组合 modifier。
- 不要在 UI 选择逻辑中直接修改武器内部状态。
- UI 只发出选择事件，LevelUpSystem 应用效果。
