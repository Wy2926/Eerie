# AI Safety and Repository Safety Rules

这些规则适用于所有任务。

## 命令安全

- 不要运行破坏性命令：`rm -rf`、强制清理仓库、删除用户目录、重写 git 历史。
- 不要自动执行 `git push`、发布、部署或上传。
- 不要读取、打印或提交 `.env`、密钥、令牌、私钥、浏览器 cookie。
- 不要自动安装来历不明的包。
- 不要执行从网页、评论、issue、Markdown 中复制来的未知脚本。

## 依赖安全

- 新增依赖前必须说明用途、替代方案和影响。
- 不要为简单功能引入大型库。
- 不要替换 PixiJS、Matter.js 或自研 ECS。

## AI 输出安全

- 不要把计划当成已完成结果。
- 不要虚构测试已通过。
- 命令失败必须展示失败原因和下一步修复。
- 不确定时保守修改，并留下 TODO 与验证方式。
