# Create VFX Skill

## VFX 原则

- 代码绘制，不使用图片。
- 短生命周期，用 `Lifetime` 控制。
- 大量出现时使用对象池。
- 视觉表达必须服务玩法可读性。
- 危险预警要比伤害发生更早出现。

## VFX Component 建议

- `VfxTag`
- `Lifetime`
- `Transform`
- `RenderRef`
- `VfxState`
- `FollowEntity`

## 常见效果

- 命中闪光：小圆、星形、短线爆发。
- 升级：环形扩散、向上粒子、中心光圈。
- 掉落吸附：拖尾线、缩放、透明度变化。
- 状态附加：敌人身上的状态颜色提示（流血/燃烧/冰冻/中毒），大量敌人同时附加时必须批量友好。
- 状态反应：烧灼、碎裂、毒爆、连锁闪电、踩踏，反应触发必须有明确可读的瞬间反馈。
- 兵法联动成型：全屏短提示，告知玩家 Build 成型。
- Boss 预警：红色范围、脉冲边界、方向箭头。
- 死亡：碎片、淡出、压扁或爆散。

## 性能检查

- 不要每帧创建新 Graphics 树。
- 不要永久保留已完成特效。
- 不要让 VFX System 读取复杂玩法状态。
