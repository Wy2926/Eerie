# 兵法（Build）系统设计

> 本文档定义游戏最核心的兵法系统设计方向，不涉及技术实现。整体定位见 `docs/GAME_DESIGN.md`；完整原始需求见 `docs/prd/01-core-vision-and-build-system.md`。

## 定位

兵法是游戏最核心的系统。玩家每次升级获得三选一兵法。

兵法不是装备、不是遗物，而是：**改变规则**。整个游戏真正的乐趣来自兵法组合。

## 设计原则

- 传统肉鸽 80% 都是数值，本游戏 80% 都是机制。
- 不是"攻击+10%"，而是"攻击方式发生变化"。
- **角色 × 武器 × 兵法 × 状态反应 = Build**，四者相互放大。

## 兵法五层分类

### 第一类：基础兵法（约20%）

保证 Build 稳定，承担保底作用，数量较少。

示例：攻击提升、攻速提升、暴击提升、生命提升、经验提升、射程提升、冷却降低、投射物+1。

### 第二类：武器改造（约30%）

改变攻击规则，而非攻击数值。

- **投射物**：穿透、折返、分裂、追踪、爆炸、跳跃、吸附、减速、旋转。
- **近战**：攻击扩大、剑气、拖尾、旋风、震地、击飞、吸怪、贯穿。

### 第三类：状态兵法（约25%）

让攻击附加状态。基础状态：流血、燃烧、冰冻、中毒、麻痹、眩晕、恐惧、击飞、破甲、虚弱、标记。

基础状态只是起点，真正的重点是 **状态反应**：

| 状态组合 | 反应 |
|---|---|
| 流血 + 燃烧 | 烧灼 |
| 冰冻 + 重击 | 碎裂 |
| 中毒 + 燃烧 | 毒爆 |
| 潮湿 + 雷电 | 连锁闪电 |
| 恐惧 + 击退 | 踩踏 |

玩家真正追求的是组合，不是数值。

### 第四类：规则兵法（约20%）

整个游戏最大的特色：修改游戏规则，而非攻击数字。

| 兵法 | 规则 |
|---|---|
| 借刀杀人 | 怪物死亡时传播其所有异常状态 |
| 越战越勇 | 生命越低，攻击越快 |
| 百战老兵 | 连续30秒未受伤，伤害翻倍 |
| 四面楚歌 | 周围敌人越多，伤害越高 |
| 临阵换将 | 获得武器时立即升级 |
| 一鼓作气 | Boss 出现时所有技能刷新 |
| 军阵 | 站立不动时攻击提升，移动后结束 |
| 穷追猛打 | 连续攻击同一目标造成真实伤害 |

### 第五类：核心兵法（约5%）

极少出现，拿到后直接决定整个 Build 方向。

| 兵法 | 效果 |
|---|---|
| 一人成军 | 失去所有副武器，主武器伤害与攻击范围大幅提高 |
| 左右开弓 | 主武器同时向两个方向攻击 |
| 百兵之王 | 每拥有一把武器，所有武器增加一种攻击方式 |
| 万箭齐发 | 所有投射物复制一次 |
| 生生不息 | 所有异常状态永久存在 |
| 百炼成钢 | 每拥有一种状态，获得新的攻击属性 |

## 兵法联动

兵法之间可以联动，联动应贯穿整个游戏，是 Build 最大的乐趣来源。

示例：先获得「燃烧」，之后获得「毒爆」，系统提示联动形成「瘴焰」——此后所有中毒敌人持续爆炸，Build 正式成型。

## Build 目标

玩家不会追求攻击最高，而是追求 Build 成型。示例流派：

- 流血流、爆炸流、控制流、连锁雷电流
- 火枪流、长枪军阵流、无限穿透流
- 剑气流、反击流、一人成军流

真正驱动玩家不断重复游戏的不是刷装备，而是：**"这一局，我还能组合出什么新的 Build？"**

## 实现方向约束（供后续开发参考）

- 兵法、状态、状态反应、联动全部数据驱动，配置放在 `src/game/balance/`。
- 升级三选一使用权重池（五层类别各占既定比例），考虑玩家已有武器/状态/兵法的协同。
- 状态与状态反应应独立成系统（StatusSystem / ReactionSystem 方向），不要散落在武器逻辑中。
