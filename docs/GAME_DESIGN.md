# Game Design Document

> 本文档定义游戏设计方向，不涉及技术实现。兵法（Build）系统详见 `docs/BUILD_SYSTEM.md`；完整产品需求按章节存放于 `docs/prd/`。

## 一句话定位

一款以 **明朝抗倭** 为背景的 **自动攻击 + 主动走位 + Roguelite Build** 俯视角肉鸽生存游戏。玩家扮演不同身份的明军，通过角色、武器、兵法（Build）、状态反应不断构筑属于自己的战斗流派，对抗源源不断的倭寇。所有素材由 PixiJS 代码程序化绘制。

游戏最大的乐趣不是刷装备，而是：**不断组合新的 Build。**

## 设计理念

1. **第一原则**：任何设计都必须服务于"自动攻击 + 主动走位 + Build构筑"这一核心循环。玩家关注走位、拉怪、吃经验、选 Build，而不是连招、技能释放、高 APM。
2. **第二原则**：不增加操作复杂度，只增加决策深度。所有成长围绕 Build 展开（Build/武器/状态/兵法/角色组合），不增加按键与主动技能。
3. **第三原则**：没有垃圾武器，只有不同 Build。所有武器平等，不存在品质与"最强武器"，任何武器通过 Build 都可以通关。
4. **第四原则**：**角色 × 武器 × 兵法 × 状态反应 = Build**。四者相互放大而非简单相加；换一把武器，整个玩法立即改变。

## 核心支柱

1. **移动生存**：玩家主要操作是移动和走位。
2. **自动攻击**：攻击行为由武器系统自动执行。
3. **构筑成长**：升级三选一兵法，形成局内 Build 路线。
4. **海量敌人**：倭寇密度逐步上升，但性能必须稳定。
5. **程序化视觉**：成熟像素风（参考 Skul: The Hero Slayer 的像素成熟度），所有角色、怪物、特效、UI 装饰都由 PixiJS 代码绘制（不引入图片）。

## 核心循环

```txt
进入战斗
  ↓
自动攻击 + 玩家主动走位
  ↓
击杀敌人，获得经验
  ↓
升级 → 三选一兵法
  ↓
Build 逐渐成型
  ↓
击败 Boss，获得奖励
  ↓
解锁新角色 / 武器
  ↓
再次挑战
```

整个游戏围绕一句话：**利用走位活下来，并打造最强 Build。**

## 角色定位

角色不是属性模板，而是真正决定玩法。角色决定：

- 核心机制。
- 初始武器。
- Build 方向。

示例方向：

| 角色 | 定位 |
|---|---|
| 锦衣卫 | 暴击、背刺、处决 |
| 神机营 | 火器、爆炸 |
| 县衙捕快 | 控制、击退 |

同样一把武器，不同角色，最终 Build 完全不同。

## 武器系统方向

- **全部共享**：任何角色都可以获得任何武器，包括其他角色的初始武器。
- **初始武器**：每个角色自带一把，之后可替换，无特殊限制。
- **取消品质系统**：不存在白/蓝/紫/金/神话，所有武器平等，区别仅在玩法。
- **武器定位**：每种武器有自己的定位（长枪=控制、火枪=爆发、弓=远程、飞刀=弹射、重锤=击飞），而不是"攻击更高"。
- **携带上限**：主武器 1 把，副武器 2~3 把；达到上限后获得新武器必须替换，迫使玩家思考 Build 组合。
- **无局外成长**：武器不升品质、不强化、不升星，成长全部来自兵法。
- **获取方式**：全部通过成就解锁（完成某角色、累计击杀、无限模式、隐藏挑战）。
- **隐藏武器**：存在需完成特殊条件才解锁的隐藏武器（如连续暴击100次、全程不用远程武器、无伤击败 Boss），不主动告知玩家。

## 实体类型

### Player

- 可移动，有生命值。
- 拥有主武器槽 1 个、副武器槽 2~3 个。
- 拥有拾取半径。
- 由 PixiJS 程序化绘制。

### Enemy（倭寇）

- 追踪玩家。
- 有生命值和接触伤害。
- 死亡掉落 XP 或奖励。
- 大量出现时必须性能友好。

### Weapon / Attack Entity

- 武器本体是配置和状态。
- 攻击可以生成 projectile、aura、slash、orbit、field。
- 命中由 DamageSystem 处理，可附加状态。

### Pickup

- XP、金币、回血、磁铁等。
- 可被磁吸。
- 通过 PickupSystem 结算。

### VFX

- 短生命周期实体。
- 用 PixiJS Graphics/Container 绘制。
- 不参与核心玩法状态。

## 升级设计

每次升级提供 **三选一兵法**。兵法不是装备、不是遗物，而是改变规则。传统肉鸽 80% 是数值，本游戏 80% 是机制。

兵法分五层（详见 `docs/BUILD_SYSTEM.md`）：

1. 基础兵法（约20%）：保底数值。
2. 武器改造（约30%）：改变攻击规则。
3. 状态兵法（约25%）：附加状态 + 状态反应。
4. 规则兵法（约20%）：改变游戏规则。
5. 核心兵法（约5%）：直接决定整个 Build。

升级选项使用权重池，不要硬编码在 UI。

## Build 目标

玩家追求的不是攻击最高，而是 Build 成型。示例流派：流血流、爆炸流、控制流、连锁雷电流、火枪流、长枪军阵流、无限穿透流、剑气流、反击流、一人成军流。

真正驱动玩家重复游戏的是：**"这一局，我还能组合出什么新的 Build？"**

## 失败和胜利

MVP：

- 玩家 HP <= 0 时失败。
- 生存满指定时间时胜利。

后续：

- 阶段 Boss。
- 成就解锁角色 / 武器。
- 无限模式与隐藏挑战。
