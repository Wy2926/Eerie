# Game Design Document

## 一句话定位

一个使用 PixiJS 程序化绘制素材的俯视角肉鸽生存游戏。玩家通过移动走位、自动攻击、收集 XP、升级选择和构筑组合，在越来越密集的怪物波次中坚持到最终时间。

## 核心支柱

1. **移动生存**：玩家主要操作是移动和走位。
2. **自动攻击**：攻击行为由武器系统自动执行。
3. **构筑成长**：升级三选一形成局内成长路线。
4. **海量敌人**：怪物密度逐步上升，但性能必须稳定。
5. **程序化视觉**：所有角色、怪物、特效、UI 装饰都由 PixiJS 代码绘制。

## MVP 范围

- 玩家移动。
- 一种自动武器。
- 一种追踪怪物。
- 怪物死亡掉落 XP。
- XP 拾取和升级。
- 升级三选一。
- 简单波次生成。
- Debug overlay。

## 局内循环

```txt
开始局内
  ↓
玩家移动躲避
  ↓
武器自动攻击
  ↓
怪物死亡掉落 XP
  ↓
玩家拾取 XP
  ↓
升级三选一
  ↓
武器/被动增强
  ↓
更高强度波次
  ↓
存活到时间结束或死亡
```

## 实体类型

### Player

- 可移动。
- 有生命值。
- 拥有武器槽。
- 拥有拾取半径。
- 由 PixiJS 程序化绘制。

### Enemy

- 追踪玩家。
- 有生命值和接触伤害。
- 死亡掉落 XP 或奖励。
- 大量出现时必须性能友好。

### Weapon / Attack Entity

- 武器本体是配置和状态。
- 攻击可以生成 projectile、aura、slash、orbit、field。
- 命中由 DamageSystem 处理。

### Pickup

- XP gem、金币、回血、磁铁等。
- 可被磁吸。
- 通过 PickupSystem 结算。

### VFX

- 短生命周期实体。
- 用 PixiJS Graphics/Container 绘制。
- 不参与核心玩法状态。

## 升级设计

升级分为：

- 新武器。
- 武器升级。
- 被动属性。
- 特殊规则修改器。

升级选项使用权重池，不要硬编码在 UI。

## 失败和胜利

MVP：

- 玩家 HP <= 0 时失败。
- 生存满指定时间时胜利。

后续：

- 10 分钟 Boss。
- 成就解锁。
- 局外成长。
