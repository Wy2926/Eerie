# PRD 第一章：游戏核心定位 & 兵法（Build）系统

> 本文档用于 AI 开发，不涉及技术实现，仅定义游戏设计。提供大的方向而非详细描述设计。
> 提炼版设计见 `docs/GAME_DESIGN.md` 与 `docs/BUILD_SYSTEM.md`。

---

## 一、游戏定位（Game Vision）

### 一句话介绍

一款以 **明朝抗倭** 为背景的 **自动攻击 + 主动走位 + Roguelite Build** 游戏。

玩家将在战场中扮演不同身份的明军，通过角色、武器、兵法（Build）、状态反应不断构筑属于自己的战斗流派，对抗源源不断的倭寇。

游戏最大的乐趣不是刷装备，而是：**不断组合新的 Build。**

---

## 二、设计理念（Design Philosophy）

### 第一原则

> 任何设计都必须服务于"自动攻击 + 主动走位 + Build构筑"这一核心循环。

玩家主要关注：走位、拉怪、吃经验、选择 Build。
而不是：连招、技能释放、QWER 操作、高 APM。

### 第二原则

> 不增加操作复杂度，只增加决策深度。

所有成长都围绕 Build 展开。增加的是 Build 组合、武器组合、状态组合、兵法组合、角色组合；而不是更多按键、更多主动技能、更复杂操作。

### 第三原则

> 没有垃圾武器，只有不同 Build。

每一种武器都拥有自己的定位。不存在 SSR 武器、神话武器、最强武器。所有武器平等，玩家通过 Build 让任何武器都可以通关。

### 第四原则

> 角色 × 武器 × 兵法 × 状态反应 = Build

不是简单相加，而是相互放大。

例如：锦衣卫 + 绣春刀 + 流血兵法 + 处决兵法 → **血狱追猎流**。

换一把武器，整个玩法立即改变。

---

## 三、游戏循环（Core Gameplay Loop）

```txt
进入战斗
  ↓
自动攻击
  ↓
玩家主动走位
  ↓
击杀敌人，获得经验
  ↓
升级 → 三选一兵法
  ↓
Build 逐渐成型
  ↓
击败 Boss，获得奖励
  ↓
解锁新角色 / 武器
  ↓
再次挑战
```

整个游戏围绕一句话：**利用走位活下来，并打造最强 Build。**

---

## 四、角色定位

角色决定：核心机制、初始武器、Build 方向。

角色不是属性模板，而是真正决定玩法。

| 角色 | 擅长 |
|---|---|
| 锦衣卫 | 暴击、背刺、处决 |
| 神机营 | 火器、爆炸 |
| 县衙捕快 | 控制、击退 |

同样一把武器，不同角色，最终 Build 完全不同。

---

## 五、武器系统

### 设计原则

- 所有武器全部共享：任何角色都可以获得任何武器，包括其他角色的初始武器（例如锦衣卫初始的绣春刀，其他角色同样可以获得）。

### 初始武器

- 每个角色进入游戏自带一把武器，之后可以替换，初始武器没有特殊限制。

### 武器品质

- 取消品质系统。不存在白/蓝/紫/金/神话。所有武器平等设计，区别仅在玩法。

### 武器定位

| 武器 | 定位 |
|---|---|
| 长枪 | 控制 |
| 火枪 | 爆发 |
| 弓 | 远程 |
| 飞刀 | 弹射 |
| 重锤 | 击飞 |

每种武器都有自己的定位，而不是"攻击更高"。

### 武器数量限制

- 主武器 1 把，副武器 2~3 把。
- 达到上限以后，获得新武器必须替换已有武器。
- 保证玩家真正思考 Build 组合，而不是全部拿。

### 武器成长

- 武器没有局外成长：不升级品质、不强化、不升星。
- 成长全部来自兵法。

### 武器获取

- 全部通过成就解锁：完成某角色、累计击杀、无限模式、隐藏挑战。

### 隐藏武器

- 游戏存在隐藏武器，不会告诉玩家，需要完成特殊条件。
- 例如：连续暴击 100 次、全程不用远程武器、无伤击败 Boss 等。

---

## 六、兵法（Build）系统

兵法即游戏最核心系统。玩家每次升级，获得三选一兵法。

兵法不是装备，不是遗物，而是：**改变规则**。

整个游戏真正的乐趣来自兵法组合。

### 设计原则

- 传统肉鸽 80% 都是数值；本游戏 80% 都是机制。
- 不是"攻击+10%"，而是"攻击方式发生变化"。

---

## 七、兵法分类（五层）

### 第一类：基础兵法（约20%）

保证 Build 稳定，承担保底作用，数量较少。

示例：攻击提升、攻速提升、暴击提升、生命提升、经验提升、射程提升、冷却降低、投射物+1。

### 第二类：武器改造（约30%）

改变攻击方式。改变的是攻击规则，不是攻击数值。

- **投射物**：穿透、折返、分裂、追踪、爆炸、跳跃、吸附、减速、旋转。
- **近战**：攻击扩大、剑气、拖尾、旋风、震地、击飞、吸怪、贯穿。

### 第三类：状态兵法（约25%）

玩家可以让攻击附加状态。

基础状态：流血、燃烧、冰冻、中毒、麻痹、眩晕、恐惧、击飞、破甲、虚弱、标记。

这些状态只是基础，真正重点是**状态反应**：

| 组合 | 反应 |
|---|---|
| 流血 + 燃烧 | 烧灼 |
| 冰冻 + 重击 | 碎裂 |
| 中毒 + 燃烧 | 毒爆 |
| 潮湿 + 雷电 | 连锁闪电 |
| 恐惧 + 击退 | 踩踏 |

玩家真正追求的是组合，不是数值。

### 第四类：规则兵法（约20%）

整个游戏最大的特色：改变规则。修改的是游戏规则，不是攻击数字。

| 兵法 | 规则 |
|---|---|
| 借刀杀人 | 怪物死亡，传播所有异常状态 |
| 越战越勇 | 生命越低，攻击越快 |
| 百战老兵 | 连续30秒未受伤，伤害翻倍 |
| 四面楚歌 | 周围敌人越多，伤害越高 |
| 临阵换将 | 获得武器，立即升级 |
| 一鼓作气 | Boss 出现，所有技能刷新 |
| 军阵 | 站立不动，攻击提升；移动后结束 |
| 穷追猛打 | 连续攻击同一目标，造成真实伤害 |

### 第五类：核心兵法（约5%）

极少出现，直接决定整个 Build。玩家拿到以后，Build 方向立即改变。

| 兵法 | 效果 |
|---|---|
| 一人成军 | 失去所有副武器，主武器伤害大幅提高，攻击范围提高 |
| 左右开弓 | 主武器同时向两个方向攻击 |
| 百兵之王 | 每拥有一把武器，所有武器增加一种攻击方式 |
| 万箭齐发 | 所有投射物复制一次 |
| 生生不息 | 所有异常状态永久存在 |
| 百炼成钢 | 每拥有一种状态，获得新的攻击属性 |

---

## 八、兵法联动

兵法之间可以联动。

示例：玩家先获得「燃烧」，之后获得「毒爆」，系统提示——兵法联动：燃烧 + 毒爆，形成「瘴焰」。以后所有中毒敌人持续爆炸，Build 正式成型。

这种联动应该贯穿整个游戏，也是 Build 最大的乐趣来源。

---

## 九、Build 目标

玩家不会追求攻击最高，而是追求 Build 成型。

示例流派：流血流、爆炸流、控制流、连锁雷电流、火枪流、长枪军阵流、无限穿透流、剑气流、反击流、一人成军流。

真正驱动玩家不断重复游戏的，不是刷装备，而是：

> **"这一局，我还能组合出什么新的 Build？"**
